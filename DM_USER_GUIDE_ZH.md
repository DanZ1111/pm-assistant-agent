# Designer Manager（DM）使用说明

这份说明适用于设计主管账号。DM 使用受限的 **Designer Portal（设计师入口）**，可以发布设计任务、分配设计师和处理误拒绝的提交，但不能查看项目成本、工厂、报价、内部日志或 PM 的完整项目页面。

## 一、注册账号

1. 请管理员进入 **Admin → Users**。
2. 管理员选择角色 **Designer Manager / 设计主管**，生成一次性邀请 PIN。
3. DM 会收到类似 `DM-ABC234` 的邀请码。每个邀请码只能使用一次。
4. 打开网站注册页：`/auth/register`。
5. 填写：
   - **Username**：登录用户名，至少 3 个字符，建议使用英文姓名或工作账号。
   - **Display Name**：页面显示姓名。
   - **Password**：至少 8 个字符。
   - **Confirm Password**：再次输入密码。
   - **Invite PIN**：输入管理员提供的 `DM-XXXXXX`。
6. 点击 **Create Account / 创建账号**。
7. 注册成功后进入 `/auth/login`，用用户名和密码登录。

如邀请码提示无效，请管理员重新生成。邀请码可能已经使用过，或生成时选择了错误的角色。

## 二、进入设计主管操作台

登录后会自动进入 **Designer Portal / 设计师入口**。

1. 点击页面上方的 **Manager operations / 主管操作**。
2. 进入 **Designer Manager Operations / 设计主管操作台**。
3. 操作台包含：
   - 创建和发布设计任务
   - 设计师名单
   - 指定设计师任务的分配
   - 已拒绝提交的恢复

## 三、创建并发布设计任务

### 1. 创建草稿

在 **创建并发布设计任务** 区域填写：

- **所属项目**：选择任务对应的项目。
- **任务标题**：用一句话说明设计目标。
- **截止日期**：期望设计师提交的日期。
- **可见范围**：
  - **所有活跃设计师**：所有设计师都能看到并参与。
  - **仅指定设计师**：只有被 DM 分配的设计师能看到。
- **Brief**：写清楚目标、背景、需要交付的方向或数量。
- **必须保留**：不能改变的产品特征、结构或品牌要求。
- **必须避免**：明确不接受的方向。
- **阻碍时间线**：如果该任务不完成会影响项目进度，请勾选。

点击 **创建草稿**。创建后任务还不会出现在普通设计师的任务列表中。

### 2. 检查并发布

1. 在同一页面找到 **待发布草稿**。
2. 再次确认任务标题、项目和截止日期。
3. 点击 **发布任务**。
4. 发布后，任务会出现在有权限的设计师入口中。

注意：每个项目同时只能有一个进行中的设计任务。如果项目没有出现在下拉菜单里，通常表示该项目已有草稿或已发布任务。

## 四、分配设计师

只有可见范围为 **仅指定设计师** 的已发布任务需要分配。

1. 在 **任务分配** 区域找到任务。
2. 从下拉菜单选择设计师。
3. 点击 **分配**。
4. 分配后，该设计师可以在自己的 Designer Portal 中看到任务。

## 五、查看任务和提交作品

DM 也可以像设计师一样：

1. 返回 **Designer Portal**。
2. 打开已发布任务，查看 Brief、必须保留、必须避免、参考资料和截止日期。
3. 在需要时上传 JPG、PNG、WebP 或 PDF；单个文件最大 20 MB。
4. 新文件会保留为新版本，不会覆盖旧版本。

## 六、恢复误拒绝的提交

如果 PM 误把设计师提交标记为拒绝：

1. 打开 **主管操作**。
2. 在 **已拒绝提交** 中找到对应记录。
3. 点击 **重新打开** 并确认。
4. 提交会恢复到待评审状态，不会修改项目时间线。

## 七、权限边界

DM 可以：

- 创建和发布设计任务
- 查看已发布任务和安全参考资料
- 分配设计师
- 上传设计提交
- 恢复误拒绝的提交

DM 不可以：

- 打开 PM 的完整项目详情页
- 修改项目、成本、工厂、报价、时间线或内部日志
- 管理用户或生成邀请码
- 使用 AI 直接修改设计任务

需要上述操作时，请联系对应 PM 或管理员。
